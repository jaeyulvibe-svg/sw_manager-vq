# 10분 데모 시나리오 — SW 자산관리 포털

작성일: 2026-07-08

핵심 스토리라인: **KISA 취약점 공지 → 승인 → 알림 → 패치 모니터링**으로 이어지는 실제 데이터 연동 플로우를 중심으로, 대시보드/자산/EOS/요청·승인/RBAC까지 훑는 구성. 상태를 실제로 바꾸는 액션(승인 버튼, 역할 토글)을 넣어 "라이브" 느낌을 살린다.

---

### 0:00–1:00 — 오프닝 & 대시보드
- **보안 대시보드** 탭으로 시작. KPI 카드(관리 대상 자산 / 취약점 자산(CVE) / 패치 적용률 / EOS 만료 자산)의 카운트업 애니메이션을 보여줌.
- `CriticalAlerts` LIVE 카드 강조: "CVE-2024-38112, Windows Server 2019 42대 영향" 같은 실제 심각도 표시 언급.
- 멘트: "지금 보시는 KPI는 전부 Supabase 실데이터 기반입니다."

### 1:00–2:00 — 자산 목록
- **자산 목록** 이동 → 카테고리 칩(OS/WEB/DB/Middleware), 상태 칩(취약점 있음/EOS 임박) 클릭해서 필터링.
- 한 자산 행 "상세" 클릭 → `AssetSlideover` 열어서 CVE-2026-0001, CVSS 9.8, OpenSSL 1.1.1w→3.0.13 업그레이드 이력 타임라인 보여줌.
- 멘트: "자산 하나하나에 취약점 이력과 패치 이력이 연결돼 있습니다."

### 2:00–3:00 — EOS 로드맵
- **EOS 로드맵** 이동 → 30/90일/6개월 통계 카드, 월별 바 차트.
- 리스크 테이블에서 **OpenSSL 3.0.x** 행(담당 정재율, 122일 남음, Critical, "긴급 검토") 짚기 — 발표자 이름이 실제 들어있어 임팩트 있음.

### 3:00–4:00 — 신규 자산 요청
- **신규 자산 요청** 이동 → 폼에 제품명/벤더 입력, 수집 모드(AUTO/SEMI_AUTO/MANUAL) 토글 클릭.
- 하단 요청 이력 테이블에서 REQ-2026-002(승인완료)/003(반려) 같은 상태 예시 보여줌.
- 멘트: "요청 등록부터 승인까지 프로세스가 트래킹됩니다."

### 4:00–5:30 — 승인 (라이브 상태 변경 데모)
- **신규 자산 요청 승인** 이동 (관리자 모드 필요).
- 대기 중인 **Redis(REQ-2026-004)** 행 클릭 → 우측 상세 패널 → **승인** 버튼 클릭.
- → 배지가 "승인완료"로 즉시 바뀌고 상단 카운트가 변하며 토스트("승인 완료") 발생 — 가장 "살아있는" 데모 포인트.

### 5:30–7:30 — ⭐ KISA 공지 → 승인 → 알림 → 패치 연동 (핵심 시연)
- **KISA 취약점 공지** 이동 → 미승인 Critical 공지(펄스 표시) 하나 선택.
- 상세 패널에서 "자동 매칭 후보"로 잡힌 영향 자산 목록 확인 (실시간 매칭 로직 `matchAssets` 언급).
- **승인 및 담당자 전달** 클릭 → 실제로:
  - Supabase에 승인 상태 기록
  - 매칭된 자산이 "확인필요"로 플래그
  - `notifications` 테이블에 알림 insert
  - 토스트 발생
- 바로 **"패치&취약점 모니터링에서 보기"** 버튼으로 이동 → 방금 승인한 CVE가 해당 자산의 "패치 요약"에 반영된 것 확인.
- 멘트: "공지 하나 승인했을 뿐인데, 자산 확인 상태·알림·패치 화면이 전부 한 번에 갱신됩니다."

### 7:30–8:00 — 알림 센터
- **알림 센터** 이동 → 방금 발생한 알림이 목록 최상단에 있는 것 확인. "바로가기" 클릭 → 패치 화면으로 재진입되는 것 보여줌 (연결 고리 마무리).

### 8:00–8:45 — 관리자 페이지 + RBAC
- **관리자 페이지** → "즉시 수집" 버튼 클릭 → 3초 로딩 애니메이션(스캐닝 연출) 시연.
- 헤더의 **역할 토글**을 "사용자 모드"로 전환 → 사이드바에서 "관리자 페이지", "승인" 메뉴가 사라지고 KISA 화면의 버튼이 "알림 수신 확인"으로 바뀌는 것 보여줌.
- 멘트: "권한에 따라 화면과 액션이 실시간으로 달라집니다."

### 8:45–10:00 — 마무리
- 다시 관리자 모드로 전환.
- **매뉴얼** 화면에서 월간 보고서 미리보기(전체 자산 14,829건 등 숫자) 짧게 보여주고 마무리.
- 클로징 멘트: "자산 등록부터 취약점 공지, 승인, 패치 반영, 알림까지 하나의 흐름으로 연결된 포털입니다."

## 팁

- 5:30~7:30 구간(KISA→패치 연동)이 가장 인상적이니 시간이 부족하면 다른 구간을 줄이고 여기에 여유를 두세요.
- 승인 버튼을 한 번 누르면 상태가 바뀌므로, 리허설 후 데모 직전에 새로고침(mock 상태 초기화 또는 다른 대기 항목 사용)해두는 걸 권장합니다.
