# 담당자 조치 현황(patch_tasks) — 예외요청/승인 설계 (2차 스코프)

## 배경

[[2026-07-14-patch-tasks-design.md]]에서 "내 조치 업무" 화면(`patch-tasks-view.tsx`)의 1차 스코프를 구현했다. 담당자가 조치예정/조치지연/조치완료를 등록할 수 있지만, "이 건은 조치가 불필요하다"고 판단될 때 처리할 방법이 없었다. 1차 스펙은 예외요청/승인, 담당자 변경, 관리자 완료 확인을 2차로 미뤄뒀다.

이 문서는 그중 **예외요청/승인**만 구현 범위로 정의한다. 담당자 변경과 완료 건에 대한 관리자 최종 승인은 여전히 범위 밖이다 — 조치완료는 지금처럼 담당자 등록만으로 종결된다.

## 상태값 확장

기존 `patch_tasks.status` check 제약(`배정됨`,`조치예정`,`조치지연`,`조치완료`)에 두 값을 추가한다.

- `예외요청` — 담당자가 "조치 불필요/보류" 사유를 `note`에 남기고 등록. **`note` 필수** — 빈 값이면 저장을 막는다.
- `예외승인` — 관리자가 예외요청을 승인해 티켓을 종결한 상태. `조치완료`(초록, success)와 구분되는 별도 색상(muted)으로 표시해 "완료된 조치"와 "면제된 건"을 혼동하지 않게 한다.

### 흐름

```
배정됨 → 조치예정 / 조치지연 → 예외요청(담당자, 사유 필수)
                                      ├─ [관리자] 승인 → 예외승인 (종결)
                                      └─ [관리자] 반려 → 조치예정으로 복귀
```

반려 시 기존 `note`(예외 사유)는 지우지 않는다 — 관리자가 왜 반려했는지 맥락 없이도 담당자가 이전에 뭐라고 적었는지 남아있어야 재요청 시 참고할 수 있다.

## 입력 검증 (`EditPanel`, `patch-tasks-view.tsx`)

- `status === "조치예정"`으로 저장 시 `due_date`가 비어있으면 저장을 막고 안내 (조치지연/예외요청은 기존처럼 선택 입력 유지).
- `status === "예외요청"`으로 저장 시 `note`가 비어있으면 저장을 막고 안내.
- `EDIT_STATUS_OPTIONS`에 `예외요청`을 추가한다 (담당자가 직접 선택 가능한 상태). `예외승인`은 관리자 액션으로만 도달하므로 담당자용 셀렉트 옵션에는 넣지 않는다.

## 관리자 액션

기존 "내 조치 업무" 화면을 그대로 쓴다(별도 화면 신설 없음).

- `isAdmin && task.status === "예외요청"`인 행의 "관리" 컬럼에 "승인"/"반려" `MiniButton` 2개를 노출한다 (그 외 행은 기존처럼 조회 전용, "-" 표시).
- 승인 → `update({ status: "예외승인" })`.
- 반려 → `update({ status: "조치예정" })` (`note`/`due_date`는 유지).
- 두 액션 모두 성공/실패 토스트, 목록 즉시 갱신.

## 필터/뱃지

- `STATUS_FILTERS`에 `예외요청`, `예외승인` 추가.
- `statusAccent`: `예외요청` → `"review"` (기존에 미사용이던 예비 accent 토큰 — "관리자 검토 대기" 의미로 사용), `예외승인` → `"muted"`.

## 파일 변경 범위

- `supabase/migrations/016_patch_tasks_exception.sql` (신규) — `patch_tasks_status_check` 제약을 drop/재생성해 `예외요청`, `예외승인` 추가.
- `lib/supabase/types.ts` — `patch_tasks.Row/Insert/Update`의 `status` 유니온에 두 값 추가.
- `components/pages/patch-tasks-view.tsx`:
  - `STATUS_FILTERS`, `EDIT_STATUS_OPTIONS`, `statusAccent`에 두 값 반영.
  - `EditPanel`에 조건부 필수 검증(기한/사유) 추가.
  - 관리자용 승인/반려 버튼 + 핸들러(`approveException`/`rejectException`) 추가.

## 미포함 (명시적으로 제외)

- 담당자 변경 기능.
- 조치완료 건에 대한 관리자 최종 승인 절차 (여전히 담당자 등록만으로 종결).
- 예외요청 반려 시 관리자 반려 사유 입력 (버튼 클릭만으로 즉시 반려).

## 테스트 계획

테스트 스위트가 없으므로 수동 검증으로 갈음한다.

- 담당자 역할 + 본인 이름 필터 선택 후 상태를 `조치예정`으로 두고 기한을 비운 채 저장 → 막히는지 확인.
- 상태를 `예외요청`으로 두고 메모를 비운 채 저장 → 막히는지 확인.
- 메모를 채워 `예외요청` 저장 → 목록에 `예외요청` 뱃지(review 색상) 표시 확인.
- 관리자 역할로 전환 → 해당 행에 "승인"/"반려" 버튼이 보이는지, 그 외 행은 "-"인지 확인.
- "승인" 클릭 → 상태가 `예외승인`(muted 색상)으로 바뀌고 담당자 쪽에서도 반영되는지 확인.
- 다른 건에서 "반려" 클릭 → 상태가 `조치예정`으로 돌아가고 기존 메모가 남아있는지 확인.
- 상태 필터에서 `예외요청`/`예외승인`으로 필터링이 정상 동작하는지 확인.
