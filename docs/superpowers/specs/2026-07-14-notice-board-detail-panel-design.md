# 공지사항 목록 개편 — NO 컬럼 + 제목 클릭 상세/편집 패널 설계 문서

날짜: 2026-07-14

## 배경 / 목표

`notice-board-view.tsx`("공지사항" nav, `notices` 테이블)의 목록 UI를 다음과 같이 개편한다.

- `구분`(category) 컬럼을 목록에서 제거하고, 대신 순번을 뜻하는 `NO` 컬럼을 맨 앞에 추가한다.
- `관리` 컬럼(상세/수정 버튼)을 없애고, 제목을 클릭하면 열리는 통합 패널에서 상세 확인·수정·삭제를 처리한다.
- 기존에 이 파일에 이미 구현돼 있던 두 개의 개별 UI — "상세"(행 아래 펼침, `expandedId`)와 "수정"(행이 폼으로 바뀜, `panel`) — 를 하나의 행 확장 패널로 통합한다.

**절대 건드리지 않는 것**: `notices` 테이블 스키마, `demo-data-view.tsx`의 리셋 대상 목록, 다른 화면(`notice-boards.tsx` 대시보드 위젯 등)의 `notices` 조회 로직, 검색/정렬/페이지네이션/다중선택/엑셀내보내기의 내부 동작 방식, "추가" 버튼으로 신규 공지를 등록하는 `NoticeFormPanel`(구분 선택 포함, DB `category`가 NOT NULL이라 유지 필요).

## 컬럼 구성

체크박스(관리자만) | **NO**(신규) | 제목 | 작성자 | 작성일 | 조회수 | 상태 — 총 6개 데이터 컬럼.

- `구분`은 `NOTICE_ALL_COLS`(컬럼설정 토글 대상)에서 제거한다. DB 컬럼(`category`)과 "추가" 폼의 구분 선택 필드는 그대로 둔다.
- `NO`는 체크박스와 마찬가지로 컬럼설정(`ColumnVisibilityMenu`)에 노출하지 않는 고정 컬럼이다. 정렬 불가(`SortTh` 미적용, 일반 `Th`).
- `상태`(일반/중요/긴급 배지)는 그대로 유지한다(이번 요구사항 범위 밖).

### NO 번호 계산 규칙

1. 현재 검색어(`query`)로 필터링된 결과 집합(`filtered`, 정렬 적용 전 단계)을 기준으로 한다.
2. 그 집합을 `created_at` 오름차순(오래된 것부터)으로 정렬해 1부터 순번을 매긴다 — 가장 최근 공지가 가장 큰 번호를 갖는다.
3. 이 순번은 공지 `id` → 번호 맵(`Map<string, number>`)으로 계산해 두고, 실제 화면에는 사용자가 고른 정렬(`sortKey`/`sortDir`)이 적용된 `filtered` 순서 + 페이지네이션(`pagination.pageItems`)으로 행을 그리되, 각 행의 NO는 이 맵에서 조회해 표시한다.
4. 따라서 페이지를 넘기거나 정렬을 바꿔도 각 공지의 NO는 변하지 않고, 검색어가 바뀌면 새 필터 결과 기준으로 다시 계산된다.

## 제목 클릭 → 행 확장 패널

기존 `expandedId`(상세 펼침)와 `panel`(수정 인라인 폼) 두 상태를 `openId: string | null` 하나로 합친다. 제목을 클릭(또는 포커스 후 Enter/Space)하면 해당 행 바로 아래에 패널이 펼쳐진다. 다른 제목을 클릭하면 열려 있던 패널은 닫히고 새 패널이 열린다(단, 아래 미저장 가드 통과 시).

제목 셀은 `<button type="button">`으로 렌더링하고, hover 시 밑줄/색상 변화로 클릭 가능함을 표시하며, 목록에서는 `line-clamp-2 whitespace-normal`(기존과 동일)로 말줄임한다.

### 패널 상태 모델

- `openId: string | null` — 펼쳐진 공지 id
- `editMode: boolean` — 열린 패널이 보기/편집 중 어느 쪽인지
- `draft: { title: string; content: string; status: NoticeStatus }` — 편집 중 임시 값
- `deleteRequest`(기존 상태 재사용) — 삭제 확인 다이얼로그 대상

### 보기 모드

- 제목, 본문(`content`, `whitespace-pre-wrap`), 작성자, 작성일, 상태 배지를 읽기전용으로 표시.
- 버튼:
  - 관리자: `[수정] [확인] [삭제]`
  - 일반 사용자: `[확인]`만

### 수정 모드 (관리자 전용, `[수정]` 클릭 시 진입)

- 제목 입력란, 본문 textarea, 상태 드롭다운(일반/중요/긴급)이 편집 가능 상태로 전환. 작성자·작성일은 계속 읽기전용.
- 버튼이 `[저장] [취소]`로 바뀐다.
- `[저장]`: 제목 공백 검증(비어있으면 저장 막고 인라인 안내) 후 `supabase.from("notices").update({ title, content, status }).eq("id", openId)`. 성공 시 토스트 성공 메시지 + 목록 새로고침 + 보기 모드로 복귀(패널은 유지, 갱신된 값 표시). 실패 시 토스트 에러 메시지, 편집 상태 유지.
- `[취소]`: draft를 원래 값으로 되돌리고 보기 모드로 복귀. dirty 여부와 무관하게 즉시 취소(별도 확인 없음 — 확인/패널전환/이탈 시에만 가드가 개입).

### `[확인]` 버튼 / 패널 닫기 공통 로직

- 수정 모드이고 draft가 원본과 달라 dirty 상태이면, 프로젝트 공용 `useUnsavedGuard().confirmLeave()`(현재 `app/page.tsx`에서 뷰 전환 시에만 쓰이던 훅을 이 화면 내부 상태 전환에도 재사용)를 호출해 `window.confirm` 안내 후 취소 시 중단.
- 통과하면 `openId`를 `null`로 만들고 보기/편집 상태 초기화.
- 다른 제목 클릭으로 패널을 전환하는 경우도 동일 가드를 거친 뒤 `openId`를 새 id로 교체한다.
- `editMode` 진입/이탈 및 draft 변경 시 `setDirty(...)`를 호출해 가드 상태를 동기화한다. 편집 중이 아니면 항상 `dirty=false`.

### `[삭제]` 버튼 (관리자 전용)

- 기존 `ConfirmDialog`(이 파일에 이미 있는 일괄삭제용 컴포넌트) 재사용. 제목을 문구에 포함: `"{title}"을(를) 삭제할까요?`.
- 확인 시 `supabase.from("notices").delete().eq("id", openId)`. 성공하면 패널 닫고(`openId=null`) 목록 새로고침 + 성공 토스트. 실패하면 에러 토스트, 패널은 유지.

## UX / 접근성

- 제목 버튼: `aria-expanded`, `aria-controls`로 펼침 상태 명시. 포커스 링 표시.
- 모바일 폭에서 패널 버튼 줄(`수정|확인|삭제` 또는 `저장|취소`)은 `flex-wrap`으로 잘리지 않게 처리.
- 삭제 버튼은 기존 프로젝트 컨벤션대로 `accent="destructive"`(붉은 톤).
- 패널 영역은 테이블 `colSpan` 전체 폭을 사용, 좁은 화면에서도 텍스트가 줄바꿈되도록 `whitespace-pre-wrap`/`whitespace-normal` 유지.

## 엑셀 내보내기

`구분` 컬럼을 내보내기 컬럼 목록에서 제거하고 `NO`를 맨 앞에 추가한다(같은 고정 rank 맵 사용).

## DB

`notices` 테이블 구조 변경 없음. `title`, `content`, `author`, `created_at`, `status`, `views` 컬럼 그대로 사용(마이그레이션 013에서 이미 `content` 존재 확인됨).

## 영향 파일

- `components/pages/notice-board-view.tsx` (거의 전체 재작성 — 컬럼 정의, 행 렌더, 패널 상태/로직)
- `components/dashboard/notice-boards.tsx` — 이번 범위 밖(대시보드 위젯의 별도 테이블, 요구사항에 없음 — 변경하지 않음)

## 검증 계획

- 관리자/일반 사용자 각각으로 공지 목록 진입 → 제목 클릭 → 보기 패널 확인.
- 관리자: 수정 → 제목/본문/상태 변경 → 저장 → 목록·패널에 반영 확인. 취소 시 원복 확인.
- 편집 중 확인 버튼 또는 다른 제목 클릭 시 `window.confirm` 발생 확인, 취소하면 편집 유지.
- 삭제 → 확인창에 제목 노출 → 삭제 후 목록에서 사라짐 확인.
- 검색어 입력 후 NO가 필터 결과 기준으로 재계산되는지, 정렬 변경/페이지 이동 후에도 각 행의 NO가 유지되는지 확인.
- 엑셀 내보내기 파일에 NO 포함·구분 제외 확인.
- 기존 기능(검색/정렬/페이지네이션/다중선택 일괄삭제/추가) 회귀 없는지 확인.
- `pnpm lint`, `pnpm build` 통과 확인.
