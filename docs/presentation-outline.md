# 발표 자료 구성안 — AI SW Asset Master

작성일: 2026-07-08

프로젝트를 처음 보는 사람이 5분 안에 이해할 수 있도록 구성한 발표 자료 초안.

## 서비스 한 줄 소개
전사 소프트웨어 자산의 **등록 → 취약점 공지 → 승인 → 패치 → EOS(단종) 관리**까지 하나의 흐름으로 연결한 AI 기반 통합 자산관리 포털

## 서비스가 해결하는 문제
- 여러 부서/서버에 흩어진 SW 자산 현황을 한눈에 파악할 수 없음
- KISA·제조사 취약점 공지를 사람이 일일이 읽고 영향 자산을 수작업으로 매핑 → 누락·지연 발생
- EOS(지원종료) 임박 자산을 놓쳐서 보안 공백이 생김
- 신규 자산 등록/승인이 메일·문서로 흩어져 추적이 안 됨
- 관리자/사용자마다 봐야 할 화면과 권한이 다른데 통제가 안 됨

## 핵심 기능 5개
1. 실시간 보안/자산 대시보드 (KPI, CVE 트렌드, Critical 알림)
2. **KISA 공지 → 자산 자동 매칭 → 승인 → 알림 → 패치 반영**까지 이어지는 연동 플로우
3. 신규 자산 요청 및 승인 워크플로우
4. EOS(단종) 로드맵 및 리스크 관리
5. 역할 기반 접근 제어(RBAC) + 관리자 전용 마스터 데이터/권한 관리

## 발표용 스크린샷 목록

| 순서 | 화면 | 캡처할 내용 | 발표 포인트 |
|---|---|---|---|
| 1 | 보안 대시보드 | KPI 카드 4개 + Critical Alerts 카드 | 실시간 데이터 기반 현황 파악 |
| 2 | 자산 목록 | 필터링된 표 + 상세 슬라이드오버(CVE 이력) | 자산별 취약점·패치 이력 추적 |
| 3 | KISA 취약점 공지 | 상세 패널의 "자동 매칭 후보" + 승인 버튼 | 공지 하나가 실제 보유 자산과 자동 매핑됨 |
| 4 | 패치&취약점 모니터링 | "패치 요약(CVE)" 컬럼 | 공지 승인 결과가 패치 화면에 자동 반영 |
| 5 | 알림 센터 | 방금 발생한 알림 + "바로가기" | 승인 → 알림이 자동으로 전파됨 |
| 6 | EOS 로드맵 | 리스크 테이블 (담당자·D-day·심각도) | 단종 임박 자산을 사전에 관리 |
| 7 | 신규 자산 요청 / 승인 | 요청 폼 + 승인 화면 상태 변경 | 워크플로우 기반 등록·승인 |
| 8 | 관리자 페이지 | 사용자 권한 테이블 + 역할 토글 | 권한별 접근 통제(RBAC) |

## 각 화면 핵심 포인트
- **대시보드**: "모든 숫자가 실데이터(Supabase) 기반"이라는 점을 먼저 강조
- **자산 목록**: 검색/필터 → 상세 슬라이드오버로 드릴다운되는 UX 흐름을 보여줄 것
- **KISA 공지**: 미승인 상태에서는 "자동 매칭 후보(잠금)"였다가 승인 후 확정되는 **전/후 비교**가 핵심
- **패치 모니터링**: KISA에서 승인한 CVE가 이 화면의 특정 자산 행에 그대로 나타나는 것을 직접 연결해서 보여줄 것
- **알림 센터**: "방금 내가 승인한 액션이 알림으로 즉시 나타난다"는 실시간성 강조
- **EOS 로드맵**: 담당자 이름이 실제로 배정돼 있다는 점(책임 소재 명확화)
- **요청/승인**: 버튼 클릭 한 번으로 배지·카운트가 즉시 바뀌는 것(라이브 데모용)
- **관리자 페이지**: 역할 토글을 바꾸는 순간 메뉴/버튼이 실시간으로 달라지는 것
